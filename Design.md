# 《灰穹回声：小空》游戏设计文档（重构版）

更新时间：2026-04-02  
适用版本：Godot 当前工程（`level_001` + `Xiaokong1`）

---

## 1. 项目定位

- 游戏类型：第一人称生存互动 + AI 对话叙事。
- 核心卖点：玩家通过自然语言与小空交流，小空不仅“说话”，还会依据状态与语义做动作反应。
- 体验目标：让“喂养/照料/对话/观察反应”形成短循环，推动关系成长与生存推进。

---

## 2. 世界观与角色

### 2.1 新世界观（完全虚构）

大灾后第 18 年，地表成为“灰穹带”，可用资源极少。玩家作为维护型仿生体 `R-7`，守着一处地下微型避难站。小空是站内唯一的人类居民。  
每天都要在“资源分配”和“情感照料”之间做选择，任何偏科都可能导致关系崩塌或生存失败。

### 2.2 角色关系

- 玩家 `R-7`：执行者与照料者，负责搜集和分配资源。
- 小空：具有独立情绪与生理需求，不是任务面板。她会记住玩家长期行为并改变互动方式。

### 2.3 资源说明（美术）

- 小空当前模型来自《碧蓝档案》风格二创资源，仅用于个人学习和原型验证。
- 世界观、剧情、系统均为原创设计。
- 对外发布版本建议替换为原创角色资源。

---

## 3. 核心玩法循环（新）

1. 避难所内经营：玩家在室内分配现有物资，和小空对话、投喂、安抚。
2. 是否外出抉择：每天可选择“留在室内”或“发起一次外出搜索”。
3. 文本外出结算：玩家不离开场景，由 AI/系统生成外出遭遇与结果。
4. 状态与库存变化：按结果更新库存（食物/水/药/零件）与小空状态。
5. 动作反馈：小空按结果触发动作树反应（欢迎、疲惫、失落、激动）。
6. 生存压力累积：不外出会持续消耗库存，迫使玩家承担外出风险。
7. 日终结算：结算“活下来 + 关系维持 + 库存安全线”，进入下一天。

### 3.1 60s 式外出搜索机制（核心）

- 外出是“文本决策事件”，不是第三人称跑图。
- 结果由 AI 决定（后端可控），本地保留 fallback 随机表保证可玩。
- 每次外出有固定成本（至少消耗食物/水），再根据风险等级决定收益和失败概率。
- 玩家可不外出，但室内生存消耗会让资源逐步枯竭。
- 关键体验是“风险选择”，不是“操作技巧”。

## 4. 场景交互设计

### 4.1 交互方式

- `E` 键交互（短按/长按）。
- 射线指向 + HUD 提示。
- 可交互对象接口统一：`get_prompt_text`、`get_interaction_time`、`interact`、`short_interact`。

### 4.2 场景对象分层

- 生存资源：水、食物、急救品、材料。
- 容器对象：箱子、储物柜（可搜刮、可保存）。
- 角色对象：小空（可对话、可投喂、可指令移动）。
- 叙事对象：照片、信件、录音终端（提供世界信息和情绪素材）。
- 外出终端：在避难所内发起“外出搜索”并查看遭遇报告。

---

## 5. 小空状态系统（玩法核心）

### 5.1 状态字段（建议标准）

- `hunger`：0-100，越低越饿（100 最饱，0 极饿）。
- `thirst`：0-100，越低越渴（100 最不渴，0 极渴）。
- `mood`：0-100，越高心情越好。
- `favor`：0-100，越高信任越高。

### 5.2 日内自然变化（建议值）

- 每游戏小时：`hunger -3`、`thirst -4`。
- 当 `hunger <= 20`：每小时 `mood -2`。
- 当 `thirst <= 20`：每小时 `mood -3`。
- 当 `mood <= 20` 持续一天：`favor -4`。
- 当 `mood >= 70` 且两项生理状态均 < 60：`favor +2`（每日上限）。

### 5.3 状态区间与行为倾向

- 0-39：稳定。更容易触发积极动作（问候、互动）。
- 40-69：普通。中性反应为主。
- 70-100：危险。更容易触发疲惫、冷淡、拒绝。

---

## 6. 食物/饮水与加成（新定义）

> 当前工程已有 `water_bottle` 资源。食物资源可按同结构扩展。

| item_id | 类型 | 即时效果 | 附加效果 | 动作倾向 |
|---|---|---|---|---|
| water_bottle | 饮水 | `thirst +30` | `mood +4`（若原本口渴<=30） | `Drinking` |
| ration_bar | 食物 | `hunger +25` | `mood +2` | `Idle` / `SittingIdle` |
| sweet_snack | 食物 | `hunger +8` | `mood +8`（短时） | `StandingGreeting` |
| hot_meal | 食物 | `hunger +35` | `mood +10`、`favor +2` | `SittingIdle` |

### 6.1 叠加规则

- 单次交互只消费一个物品条目。
- 状态值最终都 clamp 到 `0..100`。
- 同一游戏小时内重复投喂产生递减收益（防止刷好感）。

## 7. AI 对话到动作的执行链路

### 7.1 链路目标

`玩家输入 -> AI 返回 JSON -> 解析指令 -> 小空动作树执行 -> 更新状态 -> 保存`

### 7.2 当前动作树可用动作（必须对齐脚本）

来自 `scripts/xiaokong/xiaokong_animation_state_machine.gd`：

- `Idle`
- `StandingGreeting`
- `Drinking`
- `Salute`
- `Kiss`
- `SittingIdle`
- `Laying`
- `LeftTurn`
- `RightTurn`

移动相关由导航系统自动处理：

- `navigate_to(world_position)`
- 行走状态 `Walk`、转向请求 `LeftTurn/RightTurn` 由导航组件联动触发。

### 7.3 AI action 映射规范（建议）

- 未识别动作：回退 `Idle`。
- 如果返回了目标点：优先 `navigate_to`，到达后再触发非移动动作。
- 若处于坐/躺上下文且请求站立动作：先经由状态机过渡到可执行状态再触发。

## 8. 后端协议（v0.2 草案）

### 8.1 请求体（Godot -> Server）

```json
{
  "day": 3,
  "time_min": 1140,
  "npc_stats": {
	"hunger": 62,
	"thirst": 73,
	"mood": 41,
	"favor": 27
  },
  "context": {
	"scene_items": ["water_bottle", "ration_bar"],
	"player_inventory": ["water_bottle", "photo_fragment"],
	"container_items": ["water_bottle"]
  },
  "given_item": "water_bottle",
  "player_input": "先喝点水，今天辛苦了。"
}
```

### 8.2 响应体（Server -> Godot）

```json
{
  "dialogue": "谢谢……喉咙舒服多了。",
  "emotion": "relieved",
  "action": "Drinking",
  "move_target": "",
  "stat_change": {
	"thirst": 30,
	"mood": 4,
	"favor": 1
  },
  "memory_tags": ["care_received", "trusted_response"]
}
```

### 8.3 SSE 分片兼容（对齐当前 `AIManager.gd`）

流式分片字段保持：

- `dialogue_chunk`
- `full_json_so_far`
- `is_done`

## 10. 脚本落地优先级（建议开发顺序）

### P0（先打通）

- 新增 `StateComponent`：托管 `hunger/thirst/mood/favor`。
- 新增 `StockComponent`：托管避难所库存（水/食物/药品/零件）。
- 新增 `ExpeditionComponent`：发起外出、结算结果、更新库存与状态。
- 新增 `AIActionRouter`：把 AI 返回动作映射到 `trigger_action` / `navigate_to`。
- 扩展 `ItemData.gd`：增加 `consumable_effect`，实现投喂加成。

### P1（体验完整）

- 在 `level_001` 接入外出终端 UI（风险等级 + 外出文案输入 + 结果展示）。
- 接入“留守对话”按钮：不外出也能推进时间，但会吃库存。
- 接入 `DailyDirector`：每日 2 个生存事件目标（限时完成）。
- 保存/读取：小空状态、库存、当日事件、外出日志。

### P2（内容扩展）

- 多类外出事件模板（废墟、交易、伏击、天气）。
- AI 记忆标签驱动中长期关系变化。
- 外出结果与室内对话联动（比如“受伤归来”触发特殊对话分支）。

## 13. 30 天冲刺排期（建议）

### 第 1 周：把核心闭环跑起来

- 目标：可完整玩“留守/外出/结算”一天。
- 必做：
  - 接入 `StockComponent + StateComponent + ExpeditionComponent`。
  - 完成外出 fallback 结果表（先不依赖后端 AI）。
  - 完成最小 UI：按钮 + 风险等级 + 文本结果框。

### 第 2 周：把 AI 接入外出叙事

- 目标：外出结果由 AI 文本决定，fallback 兜底。
- 必做：
  - 外出请求协议：库存快照、状态、风险、玩家输入。
  - 外出响应协议：`summary/stat_change/stock_delta/action/tags`。
  - 用 `AIActionRouter` 驱动小空动作反馈。

### 第 3 周：把“好玩”做出来

- 目标：形成可感知的策略压力。
- 必做：
  - `DailyDirector` 每天 2 个限时事件（不完成有惩罚）。
  - 外出风险分层（低风险稳定、高风险高收益高惩罚）。
  - 关系阈值奖励：`favor` 达到档位后解锁对话和正向修正。

### 第 4 周：稳态与打磨

- 目标：可连续玩 7-15 天且不崩。
- 必做：
  - 存档一致性（状态、库存、事件、日志）。
  - 数值调平（避免“必死局”和“无脑最优解”）。
  - 事件文本扩写、动作触发清理、体验修 bug。
