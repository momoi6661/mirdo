# 《废土微光：铁皮与少女》 (Wasteland Glimmer) - 游戏设计与开发文档

## 一、 核心设定 (Core Concept)
- **主角 (玩家)**：战前遗留的护卫机器人（代号 R-7）。**无生存属性需求**，不需要进食和饮水。玩家以 3D 第一人称/第三人称视角在避难所内自由移动。
- **NPC (AI伴侣)**：人类幸存少女“小雅”。拥有【饱食度】、【水分】、【心情值】。
- **游戏目标**：在地下避难所中，依靠**极其有限的固定物资**，通过合理分配和语言安抚（AI聊天），让小雅存活 **15天** 等待救援。

## 二、 核心玩法循环 (Core Loop)
- **3D 沉浸式交互 (走近触发)**：
  - 平时 UI 隐藏，玩家在 3D 避难所内自由走动，观察环境和小雅的状态。
  - **交互方式**：玩家必须**走近小雅**，准星对准她时屏幕提示 `[F] 建立通讯链路`。按下 F 键后，唤起“机甲风通讯终端 UI”（整合了聊天和物资分配），此时玩家锁定移动，解锁鼠标。
- **动态时间流逝机制 (Dynamic Time Passing)**：
  - 每天的时间固定从早上 **08:00** 开始，晚上 **22:00** 结束。
  - 在通讯 UI 中操作会推进时间：**深度聊天**消耗 30 分钟，**给予物品**消耗 15 分钟。
- **POI节点式移动 (Point of Interest Movement)**：
  - 避难所为 3D 场景，小雅根据**时间日程**或**AI决策**在特定节点（Marker3D）之间移动。
  - 核心节点划分：【床铺】、【餐桌】、【浴室】。
  - 浴室隐私机制：小雅进入浴室后关门，模型隐藏并播放水声（极大地节省了动画开发成本，增强角色边界感）。
- **生理状态衰减与单日结算**：
  - 每一次聊天交互，都会轻微扣除小雅的饱食度和水分（比如扣 1 点），防止无限聊天。
  - 晚上 22:00 强制休息，进入下一天，执行单日大额状态扣除。

## 三、 Godot 端需实现的功能清单 (To-Do List)

> **开发原则：极简、好用、不搞复杂系统。**

### 1. 玩家控制器与交互 (Player.gd) [新增]
- [ ] **基础移动**：简单的 WASD 移动和鼠标视角旋转 (CharacterBody3D)。
- [ ] **射线检测**：摄像机中心挂载 `RayCast3D`，检测前方 2 米内是否有属于 `Interactable` 分组的物体（即小雅）。
- [ ] **按键交互**：检测到小雅时显示 UI 提示。按下 `F` 键，触发事件打开 UIManager，同时暂停玩家视角的转动，显示鼠标指针。

### 2. UI 与交互界面 (UIManager.gd) [修改为按需呼出]
- [ ] **默认隐藏**：游戏开始时，通讯面板是隐藏的。
- [ ] **机甲风通讯面板**：
  - **状态区**：显示天数、时间、小雅的生命体征进度条。
  - **库存区**：固定几个按钮代表物资，点击后提示“耗时15分钟移交物资”。
  - **聊天区**：打字机对话框 + 输入框 + 发送按钮。
- [ ] **关闭逻辑**：点击面板上的 `[关闭通讯]` 按钮，隐藏面板，恢复玩家的 3D 移动和视角控制。

### 3. 核心系统逻辑 (GameManager.gd)
- [ ] **状态管理**：记录天数 (Day 1-15)、时间 (08:00 - 22:00)、NPC状态 (饱食/水分/心情)。
- [ ] **时间与衰减逻辑**：每次交互后增加时间、微扣状态。22:00 触发 `next_day()`。

### 4. AI 通信模块 (AIManager.gd)
- [ ] **流式 HTTP 请求**：(已完成脚本基础) 向后端发送状态和文本，接收流式 JSON，解析 `dialogue`, `emotion`, `action`, `hunger/mood_change`。

### 5. 3D 场景与 NPC 导航 (NPCController.gd)
- [ ] **寻路与移动**：放置 `Marker3D`，使用 `NavigationAgent3D` 让小雅在床、桌子、浴室间移动。

## 四、 后端 AI 接口规范 (API Spec) - 备忘
*(此部分由后端 Python 实现，Godot 端按此标准对接)*

**Godot 发送的 JSON 格式:**
```json
{
  "day": 3,
  "time": 600, // 早上10:00 (10 * 60)
  "ai_hunger": 45,
  "ai_mood": 60,
  "player_text": "今天感觉怎么样？",
  "given_item": "过期的罐头" 
}
```

**后端返回的 JSON 格式:**
```json
{
  "dialogue": "谢谢铁皮哥，我拿着去桌子那边吃吧...",
  "emotion": "weak_smile", 
  "action": "move_to_table", // 新增：控制NPC移动的指令 (如 idle, move_to_bed, move_to_table, move_to_bathroom)
  "hunger_change": 20, 
  "mood_change": 5
}
```
