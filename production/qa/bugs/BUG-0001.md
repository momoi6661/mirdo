# Bug Report

## Summary
**Title**: 医疗柜拖拽物品到背包时，上半部格子释放被误判为世界外/取消  
**ID**: BUG-0001  
**Severity**: S2-Major  
**Priority**: P1-Immediate  
**Status**: Verified Fixed  
**Reported**: 2026-05-21  
**Reporter**: User

## Classification
- **Category**: UI / Gameplay
- **System**: 3D 背包双面板拖拽转移
- **Frequency**: Always / Often
- **Regression**: Yes

## Environment
- **Build**: 本地 Godot 项目 `D:\AAgodot\FPS`
- **Platform**: Windows / Godot 4.6.2
- **Scene/Level**: 地堡内医疗柜双面板交互
- **Game State**: 打开医疗柜，左侧/上方为医疗柜面板，右侧/下方为玩家背包面板

## Reproduction Steps
**Preconditions**: 医疗柜中有药品，玩家打开医疗柜双面板。

1. 从医疗柜面板拖拽药品。
2. 将药品拖到玩家背包面板格子的上半部或靠上边缘。
3. 松开鼠标。

**Expected Result**: 药品应转移到玩家背包目标格，或在背包面板上自动进入可用格。  
**Actual Result**: 松手位置被误判为世界外/来源面板空白区域，拖拽取消；只有拖到背包格子中间偏下时才成功。

## Technical Context
- **Affected files**:
  - `controllers/interaction/holo_inventory_panel_3d.gd`
  - `controllers/interaction/holo_inventory_dual_panel_3d.gd`
  - `tests/system/test_holo_inventory_panel_drag_transfer.gd`
- **Related systems**:
  - `HoloInventoryPanel3D.transfer_requested`
  - `HoloInventoryDualPanel3D._resolve_release_target()`
  - `InventoryTransferService.transfer_between_storages()`
- **Root cause**:
  1. 来源面板命中自身非格子区域时，原逻辑直接 `_cancel_drag()`，没有把释放交给双面板转移判断。
  2. 双面板释放目标过度依赖物理盒体射线命中；面板有厚度和倾角时，命中点可能偏离真实格子平面，导致上半部/边缘区域识别不稳定。

## Fix Record
**Fixed**: 2026-05-21  
**Resolution**:
- 来源面板在自身空白区域释放时改为 `_release_drag_outside()`，继续发出 `transfer_requested`。
- 双面板释放目标增加面板平面兜底判定。
- 物理命中点会投影回格子平面后再解析 slot。

**Regression tests**:
- `tests/system/test_holo_inventory_panel_drag_transfer.gd`
  - 覆盖 `allow_release_outside_panel=false` 时仍发转移请求。
  - 覆盖来源面板空白区域释放不吞掉转移。
  - 覆盖格子上半部/边缘释放可解析到正确 slot。
