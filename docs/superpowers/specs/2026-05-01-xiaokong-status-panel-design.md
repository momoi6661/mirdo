# Xiaokong Status Panel Design

**Date:** 2026-05-01

## Goal

为小空的 `查看` 交互补上一套正式可复用的状态查看面板：
- 面板为 **挂在小空身上的 3D 浮空面板**。
- 内容只显示 **4 项核心状态条** 与 **当前命中的纯文字状态**。
- 所有状态规则统一收进 `Resource`，后续主要通过配表扩展。
- 打开状态面板后，原交互选项面板让位，不与状态面板同时堆叠。
- 顺手修正现有交互选项面板的选中态：**切换时字号/缩放差异要更明显**。

## Core Requirements

1. **状态面板表现形式**
   - 玩家看到的是 3D 浮空面板。
   - 面板内部排版使用 2D UI 内容层，但最终承载在 3D 面板上。
   - 不是普通 HUD，也不是纯 `Label3D` 手摆文本。

2. **面板锚点归属**
   - 状态面板位置不写死。
   - 小空场景内挂一个专用 `Marker3D`，作为状态面板的显示锚点。
   - 实际位置、旋转、轻微倾斜都由该 `Marker3D` 决定，后续在编辑器直接可调。

3. **显示内容范围**
   - 只保留 4 个核心值：`饱食度 / 水分 / 心情 / 好感度`。
   - 不显示体力。
   - 不显示数字、百分比、解释段落、debug 文本。
   - 下方只显示当前命中的状态文字列表。

4. **状态展示原则**
   - 状态没有优先级。
   - 状态不额外按类型排序。
   - 命中了就显示，显示顺序严格跟随 `Resource` 配置顺序。
   - 状态文本为纯文字，不加图标、不加胶囊背景、不加分类标题。

5. **视觉风格**
   - 主色走亮一些的蓝白色。
   - 简约、可爱、轻一点，不要 debug 感。
   - 不要厚边框、复杂装饰、重科技感。
   - 状态条清楚，状态文字清楚，整体不要太抢戏。

6. **交互关系**
   - 玩家通过小空交互选项中的 `查看` 打开状态面板。
   - 状态面板打开后，角色交互选项面板隐藏。
   - 状态面板自身不负责提供选项，不抢占对话/喂食逻辑。

7. **交互选项面板同步修正**
   - 当前 world interaction 选项切换时的大小差异不明显。
   - 本次实现中需要同步加大选中态与未选中态的视觉差异。
   - 重点是“更容易一眼看出当前选中的选项”，不靠额外说明文字实现。

## Architecture

### 1. 交互入口层
沿用现有 `D:\AAgodot\FPS\components\xiaokong_character_interactable_component.gd`：
- `查看` 仍使用 `OPTION_ID_VIEW_STATUS`。
- 继续发已有全局信号 `xiaokong_status_requested`。
- 该组件只负责发起请求，不负责渲染状态面板。

### 2. 状态显示层
新增/重做一套小空状态显示面板：
- 面板归属于小空，不归属于玩家。
- 面板场景挂在小空附近，读取专用 `Marker3D` 作为锚点。
- 面板脚本负责：显示、隐藏、绑定状态源、刷新 4 条状态、刷新状态列表。

### 3. 状态规则层
新增统一总 `Resource`：
- 保存状态规则数组。
- 数组顺序即最终显示顺序。
- 面板只读取规则，不把具体状态判断硬编码进 UI。

### 4. 现有交互选项面板微调
沿用现有 `D:\AAgodot\FPS\controllers\interaction\world_interaction_panel_component.gd`，仅做选中态强化：
- 提高当前选项的缩放/抬升/高亮差异。
- 不改动其现有交互职责边界。
- 该项属于同批 UI polish，不独立拆系统。

## Data Model

### Core Stat Source
继续读取小空状态组件中的 4 个字段：
- `hunger`
- `thirst`
- `mood`
- `favor`

这些值是状态面板唯一的基础数值来源。

### Rule Set Resource
建议新增一个总规则资源，例如概念上：
- `CharacterStatusRuleSetResource`

其职责：
- 存放一组状态规则。
- 控制最终显示顺序。
- 让以后加状态主要改资源，不改 UI 脚本。

### Single Rule Shape
每条规则至少需要：
- `id`
- `display_text`
- `enabled`
- `conditions`

### Condition Scope
首版条件只需要覆盖当前 4 个核心值：
- 饱食度阈值
- 水分阈值
- 心情阈值
- 好感度阈值

同时一开始就支持**多条件同时成立**，例如：
- 饱食度低 + 水分低 -> `状态欠佳`
- 心情高 + 好感度高 -> `愿意亲近你`

## UI Layout

### Top Block
上半部分固定 4 行状态条：
1. 饱食度
2. 水分
3. 心情
4. 好感度

每行只保留：
- 左侧名称
- 右侧条形值

不显示：
- 数字
- 百分比
- 额外说明

### Bottom Block
下半部分为纯文字状态列表：
- 一行一条
- 只显示当前命中的状态
- 没命中的状态不占位
- 不额外加分类头

### Motion / Presence
- 面板打开时做轻微淡入或轻微缩放进入即可。
- 面板跟随小空时允许轻微 lerp，避免硬抖。
- 关闭时用轻微淡出即可。
- 不做复杂翻页、扫光、大片装饰动效。

## Data Flow

1. 玩家对准小空，打开交互选项。
2. 选择 `查看`。
3. `XiaokongCharacterInteractableComponent` 发出 `xiaokong_status_requested`。
4. 小空状态面板监听到请求后打开。
5. 面板绑定目标小空状态组件，读取 4 个基础值。
6. 面板遍历状态规则 `Resource`。
7. 命中的规则按资源顺序加入列表并显示。
8. 状态面板显示期间持续刷新数值条与命中状态。
9. 面板关闭时取消显示，但不改动角色状态本身。

## Open / Close Rules

### Open
- 仅通过小空 `查看` 选项进入。
- 打开后，原交互选项面板隐藏。

### While Open
- 默认不锁移动。
- 默认不锁视角。
- 主要是查看，不引入背包那类复杂鼠标拖拽逻辑。

### Close
出现以下情况时关闭：
- 玩家主动关闭当前查看。
- 玩家离开有效交互范围。
- 小空对象失效、隐藏或不可用。
- 被其他更高优先级的专用面板替代。

## Error Handling

1. **状态组件未绑定**
   - 面板不报错刷屏。
   - 直接不显示内容或保持安全空态。

2. **规则资源缺失**
   - 4 条状态条仍可显示。
   - 状态文字区显示为空，不阻断面板打开。

3. **规则条件字段不完整**
   - 跳过异常规则。
   - 不因为单条坏规则导致整面板失效。

4. **锚点缺失**
   - 允许回退到小空默认节点附近，或直接不显示并给开发期警告。
   - 不能在运行期抛出致命错误。

## Verification Goals

1. 对准小空时仍能看到 `对话 / 查看` 等交互选项。
2. 选择 `查看` 后，交互选项面板隐藏，状态面板单独出现。
3. 状态面板显示位置由小空身上的 `Marker3D` 控制。
4. 状态面板只显示 4 条核心状态条，不显示数字。
5. 状态文字列表按 `Resource` 顺序稳定显示全部命中状态。
6. 删除或清空状态规则资源时，状态条仍能安全显示。
7. 状态组件短暂失效时不会报错刷屏。
8. 现有交互选项面板切换选中项时，大小差异明显强于当前版本。

## Out of Scope

- 这次不做状态页分页。
- 不做体力系统回归。
- 不做状态数值编辑功能。
- 不把状态面板和对话输入框合并成一个 UI。
- 不做新的剧情系统或关系数值结算逻辑。

## Notes

- 旧 `XiaokongStatusPanel` 仅视为 debug 初版，不保留其原表现形式。
- 本设计明确把“状态显示规则”从 UI 脚本中拆出到 `Resource`。
- 本设计允许未来继续扩展更多状态词，但首版只围绕当前 4 项核心值建立规则。
